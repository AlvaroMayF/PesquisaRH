GMUD 02 - Gerenciamento de Mudanças

Portal RH - Relatório de Alterações Implementadas

Data da Geração: 10 de Julho de 2025

1. Título da Mudança:

Padronização Global da Interface, Carga de Dados Demográficos e Validação da Funcionalidade de Múltiplas Pesquisas.

2. Descrição Detalhada da Mudança:

Esta GMUD documenta uma série de atualizações evolutivas e de padronização em todo o Portal RH. O trabalho focou em unificar a experiência do usuário (UI/UX) através da implementação de uma nova fonte e um layout de navegação padrão. Adicionalmente, concluiu-se a carga de dados demográficos essenciais dos colaboradores e realizou-se a validação completa da funcionalidade de múltiplas pesquisas, confirmando sua robustez e prontidão para uso.

3. Alterações Implementadas:

3.1. Padronização da Interface de Usuário (UI/UX):

  • Objetivo: Criar uma identidade visual coesa e profissional em todas as páginas do portal.
  • Tipografia: A fonte padrão do sistema foi alterada de 'Inter' para 'Source Sans 3'. A alteração foi feita no arquivo base.css e nos links do Google Fonts em todos os templates HTML.
  • Barra de Navegação (Navbar): A estrutura da navbar foi padronizada para um layout de 3 colunas (logo/tema à esquerda, saudação ao centro, menu à direita). A lógica de exibição do menu "Administrador" foi unificada em todos os templates.
  • Rodapé (Footer): Conforme nova diretriz de design, o rodapé foi removido de todas as páginas do sistema para uma interface mais limpa e focada no conteúdo.
  • Arquivos HTML Padronizados:
    • analitico.html
    • NovoComunicado.html
    • gerenciar_feriados.html
    • beneficios.html
    • comunicados.html
    • lista_pesquisas.html
  • CSS Refatorado: Arquivos CSS específicos de páginas (ex: comunicados.css) foram limpos para remover regras duplicadas que agora são herdadas do base.css.

3.2. Carga de Dados e Sincronização de Colaboradores:

  • Objetivo: Popular o banco de dados com as informações demográficas completas e atualizadas dos colaboradores, recebidas do DP.
  • Backend (scripts/load_colaboradores.py): O script foi corrigido para apontar para o nome de arquivo Excel correto. Sua lógica de UPSERT (INSERT/UPDATE) foi executada com sucesso.
  • Banco de Dados (SQL): A importação foi verificada com o comando SELECT, confirmando que as colunas setor, cargo e unidade na tabela colaboradores foram corretamente preenchidas.

3.3. Validação da Funcionalidade "Múltiplas Pesquisas":

  • Objetivo: Garantir o funcionamento completo do fluxo de seleção e resposta de pesquisas distintas por um mesmo colaborador.
  • Teste de Ponta a Ponta: Foi executado um teste completo que validou:
    1. A exibição correta da lista de pesquisas ativas.
    2. A exibição do formulário com as perguntas específicas da pesquisa selecionada.
    3. O envio e salvamento bem-sucedido das respostas.
    4. A atualização de status da pesquisa para "Respondido" na interface.
    5. O bloqueio que impede um usuário de responder a mesma pesquisa duas vezes.
  • Verificação no Banco de Dados: Confirmação de que as tabelas responses (com dados demográficos), response_answers e colaborador_survey_status foram todas populadas corretamente após o teste.

4. Impacto da Mudança:

  • Positivo:
    • Experiência de usuário consistente e profissional em todo o sistema.
    • Base de dados de colaboradores enriquecida, habilitando futuras análises estratégicas.
    • Funcionalidade de múltiplas pesquisas 100% validada e pronta para uso.
    • Código mais limpo e de fácil manutenção com a remoção de CSS duplicado.
  • Negativo: Nenhum impacto negativo direto esperado.

5. Plano de Rollback (Reversão):

Em caso de problemas, as alterações podem ser revertidas através do sistema de controle de versão (Git). As alterações no banco de dados exigem restauração de backup ou execução de um script de limpeza manual, se necessário.

6. Testes Realizados:

  • Testes visuais em todas as páginas padronizadas.
  • Validação do script de importação de dados e verificação dos resultados no banco de dados.
  • Execução de um roteiro de teste completo (end-to-end) para a funcionalidade de múltiplas pesquisas, incluindo a verificação final da integridade dos dados salvos.
